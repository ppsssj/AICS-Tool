# AI Workspace Activity Examples

AI 워크스페이스에서 바로 테스트할 수 있는 예시 요청 모음입니다.
각 예시는 "무슨 말을 입력하면", "어떻게 응답하면 좋은지"를 기준으로 정리했습니다.

## 1. 프로젝트 열기

입력 예시:
- `Alpha 프로젝트 열어줘`
- `최근에 보던 프로젝트로 이동해줘`
- `문서 많은 프로젝트 찾아줘`

기대 동작:
- 프로젝트가 하나로 명확하면 해당 프로젝트 컨텍스트를 활성화
- 여러 개가 비슷하면 짧은 확인 질문 반환
- 못 찾으면 후보나 다음 행동 제안

예상 응답 예시:
```json
{
  "message": "Alpha 프로젝트로 이동할게요.",
  "action": {
    "type": "navigate",
    "projectId": "proj-alpha",
    "documentId": null,
    "path": "/projects/proj-alpha"
  },
  "suggestions": [
    "작업 보드 열기",
    "문서 보기"
  ]
}
```

## 2. 현재 프로젝트 기준 작업 보기

입력 예시:
- `현재 프로젝트 작업 보드 보여줘`
- `이 프로젝트 열린 작업 보여줘`
- `리뷰 대기 작업만 보고 싶어`

기대 동작:
- 활성 프로젝트가 있으면 그 컨텍스트를 우선 사용
- 작업 보드 경로로 이동하거나, 먼저 이동 의도를 설명
- 프로젝트가 없으면 어떤 프로젝트를 볼지 짧게 질문

예상 응답 예시:
```json
{
  "message": "현재 프로젝트 작업 보드로 이동할게요.",
  "action": {
    "type": "navigate",
    "projectId": "proj-alpha",
    "documentId": null,
    "path": "/projects/proj-alpha/tasks"
  },
  "suggestions": [
    "리뷰 작업만 보기",
    "마감 임박 작업 보기"
  ]
}
```

## 3. 문서 찾기

입력 예시:
- `현재 프로젝트 문서 보여줘`
- `Alpha 프로젝트 문서 목록 열어줘`
- `기획서 문서 있는 프로젝트 찾아줘`

기대 동작:
- 현재 프로젝트 문서 화면으로 이동하거나
- 프로젝트가 명시되면 해당 프로젝트 기준으로 이동
- 애매하면 프로젝트 또는 문서명 확인 질문

## 4. 일정 확인

입력 예시:
- `이 프로젝트 일정 보여줘`
- `Alpha 프로젝트 다음 일정 알려줘`
- `이번 주 일정 있는 프로젝트 찾아줘`

기대 동작:
- 프로젝트 스케줄 화면으로 이동
- 프로젝트 문맥이 없으면 최근 프로젝트 또는 후보 제안

예상 응답 예시:
```json
{
  "message": "Alpha 프로젝트 일정 화면으로 이동할게요.",
  "action": {
    "type": "navigate",
    "projectId": "proj-alpha",
    "documentId": null,
    "path": "/projects/proj-alpha/schedule"
  },
  "suggestions": [
    "이번 주 일정 보기",
    "마감 일정 확인"
  ]
}
```

## 5. 프로젝트 탐색 질문

입력 예시:
- `최근에 작업한 프로젝트 뭐였지?`
- `문서가 가장 많은 프로젝트 알려줘`
- `열린 작업이 많은 프로젝트 보여줘`

기대 동작:
- 최근 프로젝트나 요약 정보 기반으로 짧게 답변
- 하나를 특정할 수 있으면 이동 제안
- 애매하면 선택 후보 제공

## 6. 모호한 요청 처리

입력 예시:
- `그 프로젝트 열어줘`
- `거기 문서 보여줘`
- `이거 일정 확인해줘`

기대 동작:
- 활성 프로젝트가 있으면 그 프로젝트로 해석
- 활성 컨텍스트가 없으면 바로 추정하지 말고 짧게 확인 질문

예상 응답 예시:
```json
{
  "message": "어떤 프로젝트를 말씀하시는지 확인이 필요해요. 프로젝트 이름을 알려주세요.",
  "action": {
    "type": "none",
    "projectId": null,
    "documentId": null,
    "path": null
  },
  "suggestions": [
    "Alpha 프로젝트",
    "최근 프로젝트 보기"
  ]
}
```

## 7. 데모용 추천 프롬프트

발표나 테스트에서 바로 쓰기 좋은 짧은 예시:
- `Alpha 프로젝트 열어줘`
- `현재 프로젝트 작업 보드 보여줘`
- `현재 프로젝트 문서 목록 보여줘`
- `이 프로젝트 일정 열어줘`
- `최근 프로젝트 다시 열어줘`
- `문서가 가장 많은 프로젝트 찾아줘`
- `리뷰 대기 작업 많은 프로젝트 알려줘`

## 8. 좋은 프롬프트 작성 팁

- 프로젝트 이름을 같이 쓰면 정확도가 높아집니다.
- `현재 프로젝트`, `이 프로젝트` 같은 표현은 활성 컨텍스트가 있을 때 유리합니다.
- 한 번에 하나의 의도를 넣는 편이 응답 품질이 좋습니다.
- 이동 요청인지, 조회 요청인지 짧게 분명히 쓰는 편이 좋습니다.
